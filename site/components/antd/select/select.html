url
设置url属性动态加载下拉数据，reload方法重新加载
<template>
    <w-select
        ref="selectRef"
        v-model:value="modelValue"
        url="select/list"
        style="width: 30%"
        placeholder="请选择"
    />
    <span> 选择结果：{{ modelValue }} </span>
    <a-button @click="reload">重新加载</a-button>
</template>
<script lang="ts" setup>
import { ref } from "vue";
const modelValue = ref();
const selectRef = ref();
const reload = () => {
    selectRef.value.reload();
    modelValue.value = undefined;
};
</script>

属性 ​

参数说明类型默认值
url动态加载 select 数据。低于 options 的权重string
method动态加载 select 数据的请求方法stringget
isBody动态加载 select 数据时，是否以body携带参数Booleanfalse
params动态加载 select 数据时，请求参数Object
fieldNames替换 select 中 label,value,children 字段为 select 中对应的字段Object {children:'children', label:'label', value: 'code' }
labelInValue是否返回对象，当后端需要名称和编码等的时候开启该字段，并设置如下的labelField和valueFieldBooleanfalse
labelField名称接收字段，支持双向绑定v-model:labelFieldString ,Number
valueField名称接收字段，支持双向绑定v-model:valueFieldString, Number
extField名称接收字段，支持双向绑定v-model:extField 额外的字段String, Number
extNames对应extField绑定的字段名，获取的select数据中需要获取的字段名Stringid

方法 ​

方法名描述参数
reload重新触发w-select url模式下的请求-

事件名说明回调
onDataChange操作下拉之后会触发该事件选中的行数据对象

FAQ ​

外部额外使用了change

labelInValue设置为true。且绑定labelField和valueField，但是切记不可绑定value和change事件。绑定value会造成无法回填，绑定change会照成无法返回对应字段，所以将change替换为onDataChange
